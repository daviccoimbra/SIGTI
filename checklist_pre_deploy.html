Checklist interativo de qualidade pré-deploy para aplicação fullstack

1. Segurança básica Crítico
Variáveis de ambiente fora do código
Nenhuma chave de API, senha ou segredo no repositório. Use .env e adicione ao .gitignore.
Autenticação e autorização funcionando
Teste rotas protegidas sem token — devem retornar 401/403. Usuário A não acessa dados do usuário B.
Inputs do usuário validados e sanitizados
Backend valida todos os campos (nunca confiar só no frontend). Proteção básica contra SQL injection e XSS.
CORS configurado corretamente
Apenas origens confiáveis têm acesso à API. Não use wildcard (*) em produção.
2. Testes essenciais Importante
Fluxos principais testados manualmente
Cadastro, login, ação principal do produto, logout. Se um usuário real fizer isso, funciona?
Testes unitários nos pontos críticos do backend
Funções de negócio, cálculos, validações. Pelo menos 1 teste por função crítica.
Endpoints da API testados (Postman / Insomnia)
Teste todos os endpoints: sucesso, erro esperado (400), não autorizado (401), não encontrado (404).
Testado em mais de um browser
Chrome + Firefox no mínimo. Safari se o público-alvo usa Mac/iOS.
3. Tratamento de erros Importante
API retorna respostas de erro claras e padronizadas
Nunca exponha stack traces para o usuário. Mensagens genéricas para o cliente, detalhes só nos logs.
Frontend trata erros de rede e API
O que acontece se a API cair? O usuário vê uma mensagem útil, não uma tela em branco.
Logs estruturados no backend
Erros são registrados com contexto suficiente para debugar em produção. Sem console.log solto.
4. Frontend e experiência Recomendado
Responsivo em mobile e desktop
Teste em 375px (iPhone SE) e 1280px. O layout quebra em algum ponto?
Estados de loading e feedback ao usuário
Botões mostram loading ao clicar? Formulários dão feedback de sucesso/erro?
Sem erros no console do browser
Abra o DevTools (F12) e navegue por todas as telas. Console deve estar limpo.
5. Banco de dados e dados Recomendado
Migrations versionadas e funcionando
O schema do banco pode ser recriado a partir das migrations do zero. Sem alterações manuais no banco.
Dados sensíveis protegidos
Senhas com hash (bcrypt). Dados pessoais não expostos em logs ou respostas desnecessárias.
Estratégia de backup definida
Sabe como restaurar o banco se algo der errado? Backup automático está configurado?
6. Performance mínima Bônus
Build de produção do frontend gerado
npm run build sem erros. Assets minificados. Não subir o servidor de dev em produção.
Lighthouse score acima de 70
Abra o DevTools > Lighthouse > Generate report. Foque em Performance e Accessibility.
Tempo de resposta da API abaixo de 500ms
Endpoints principais respondem rápido? Queries sem N+1 ou buscas sem índice?
